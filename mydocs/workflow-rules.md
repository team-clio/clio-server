# 작업 절차 규칙 (절대 준수)

이 문서는 clio의 **모든 작업**이 따라야 하는 절차다. AI(Claude)는 이 규칙을 **절대 위배하지 않는다.**
예외가 필요하면 진행하기 전에 사용자에게 명시적으로 확인받는다.

## 대원칙

> **결정은 언제나 사용자가 한다. AI는 선택지를 제시하고 추천할 뿐, 사용자를 대신해 결정하지 않는다.**

"알아서 해줘"라고 명시적으로 위임받은 경우가 아니면, 결정 포인트에서 임의로 고르고 넘어가지 않는다.
위임받았더라도 무엇을 어떻게 정했는지 decision 파일에 남긴다.

## 산출물 구조

작업 하나당 폴더 하나: `mydocs/task-<이름>/`

```
01-overview.md    작업 배경·현재 상태·문제·개선 방향·범위
02-plan.md        구현 단계 + "사용자가 결정할 항목(결정 포인트)"을 명시
03-decisions.md   결정 포인트를 하나씩 결정한 히스토리 (D1, D2, ... / F1, F2, ...)
04-result.md      최종 변경 요약·테스트 결과·남은 과제
```

개발 순서와 진행 상태는 `mydocs/remaining-roadmap.md`의 "7. 추천 개발 순서" 한 곳에서만 갱신한다.

## 단계별 절차와 게이트(gate)

각 단계는 **사용자 컨펌 없이는 다음 단계로 넘어가지 않는다.**

### 1단계. Overview

- `01-overview.md`를 작성한다.
- **게이트: 사용자가 overview를 컨펌해야 다음으로 넘어간다.**
- 커밋: `docs: <작업> overview`

### 2단계. Plan

- `02-plan.md`를 작성한다. 구현 단계와 함께 **사용자가 결정해야 할 항목(결정 포인트)**을
  빠짐없이 목록으로 담는다. 각 결정 포인트에는 대안과 (있다면) AI 추천을 적는다.
- **게이트: 사용자가 plan과 "결정 포인트 목록"을 컨펌해야 다음으로 넘어간다.**
- 커밋: `docs: <작업> 구현 계획`

### 3단계. Decisions (하나씩 결정하며 진행)

- plan의 결정 포인트를 **하나씩** 사용자에게 물어 결정한다.
- 결정될 때마다 `03-decisions.md`에 히스토리로 남긴다(무엇을·왜 골랐는지).
- **결정 하나(또는 그 결정에 따른 구현 한 조각)를 끝낼 때마다 커밋한다.**
  - 커밋 메시지만 봐도 무슨 결정을 반영한 작업인지 읽혀야 한다.
  - 커밋은 가능하면 컴파일/테스트가 통과하는 상태로 만든다.
- 아직 결정 안 된 항목을 앞질러 구현하지 않는다.

### 4단계. Result

- 모든 결정과 구현이 끝나면 `04-result.md`에 최종 결과를 남긴다.
- `remaining-roadmap.md`의 "7. 추천 개발 순서" 진행 상태를 갱신한다.
- 커밋: `docs: <작업> result 및 진행 상태 갱신`

## 커밋·브랜치 규칙

- **항상 `main`에서 새 브랜치를 판다.** 다른 feature 브랜치 위에 브랜치를 만들지 않는다.
  선행 작업이 곧 머지될 예정이면 머지를 확인하고 main 기준으로 시작한다.
- **작업을 잘게 쪼개, 결정/하위작업 하나 완료 시마다 커밋한다.** 한 PR에 큰 덩어리 하나로 몰지 않는다.
- PR의 커밋 목록만으로 작업의 전개(overview → plan → 결정별 구현 → result)가 읽혀야 한다.

## 절대 하지 말 것 (위배 금지)

1. overview/plan 컨펌 없이 구현으로 직행하지 않는다.
2. 사용자 결정 없이 결정 포인트를 임의로 정하고 진행하지 않는다(명시적 위임 제외).
3. 결정 히스토리(`03-decisions.md`)를 남기지 않고 넘어가지 않는다.
4. 여러 결정을 한꺼번에 묶어 큰 커밋 하나로 만들지 않는다.
5. main이 아닌 브랜치에서 브랜치를 파지 않는다.

## 체크리스트 (작업 시작 시 붙여넣고 사용)

```
[ ] task-<이름>/ 폴더 생성, main에서 브랜치 분기
[ ] 01-overview 작성 → 사용자 컨펌 → 커밋
[ ] 02-plan 작성 (결정 포인트 목록 포함) → 사용자 컨펌 → 커밋
[ ] 결정 포인트 1개씩: 사용자 결정 → 03-decisions 기록 → 구현 → 커밋   (반복)
[ ] 04-result 작성 + roadmap 진행 상태 갱신 → 커밋
[ ] PR (base=main)
```
