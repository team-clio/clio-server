# LLM 근거 검증 결정 히스토리

각 항목: **후보 → 선택 → 이유**. plan의 결정 포인트 E1~E7.
전제: 참조 검증(언급한 파일/클래스가 근거에 존재하나)이지 의미 검증이 아니다. 최대 리스크는 오탐.

---

## E1. 허용 근거 집합

**후보**
- (a) relatedCode의 filePath(+basename)·symbolName만.
- (b) 위 + flows의 className·filePath.
- (c) 위 + 도메인·검색어.

**선택: (b) relatedCode + flows.**

**이유**: 흐름 클래스도 #10에서 grounding 입력으로 LLM에 줬으니 언급이 정당 → 허용에 넣어야 오탐↓.
도메인·검색어(c)는 코드 식별자가 아니라 불필요. 허용 집합은 넉넉히(오탐↓ 방향).

---

## E2. 언급 추출 규칙

**후보**
- (a) 공격적: 대문자 시작 단어·경로·점 포함 토큰 다 추출.
- (b) 보수적: `*.java` + multi-hump CamelCase(대문자 언덕 2개+)만.

**선택: (b) 보수적.** 매칭은 대소문자 무시·관대(애매하면 통과).

**이유**: 단일 대문자 단어(`Payment`)·소문자·한글은 추출 안 함 → 확실한 코드 식별자만 검사해 오탐 최소.
"오탐으로 신뢰 깎느니 미탐(교묘한 환각 놓침) 감수". 명백한 환각만 잡는 1차 방어임을 result에 명시.

---

## E3. 위반 시 액션 (중대)

**후보**
- (a) 경고만: LLM 텍스트 유지 + 위반 목록 표시.
- (b) 문장·필드 제거(redact).
- (c) 전체 rule-based 폴백.

**선택: (a) 경고만.**

**이유(사용자 선택)**: 리포트를 보존하면서 근거 없는 언급을 **드러내** 사람이 판단하게 한다. roadmap
"제거하거나 **경고**한다"의 경고 쪽을 택함(완료 기준 충족: 경고로 "근거에 묶임"을 가시화).
- (c) 전체 폴백 대비: 좋은 리포트를 오탐 1개로 통째로 버리지 않음(보수적 추출이라도 폴백은 손실 큼).
- (b) redact 대비: 문장 도려내 문맥 깨지 않음.
- **함의**: 경고를 실제로 노출해야 의미가 있다 → **E4(노출 방법)가 로그만이면 안 되고 사용자에게 보여야 함.**

---

## E4. 경고 노출 방법

**후보**
- (a) 응답에 `evidenceWarnings` 필드 신설.
- (b) `rationale`에 append.
- (c) 로그만 (E3=경고만과 상충 — 사용자가 못 봄).

**선택: (a) `evidenceWarnings` 필드 신설.**

**이유(사용자 선택)**: 위반 목록을 구조화 리스트로 응답에 추가. #8 similarIssues·#9 relatedDecisions와
동일 패턴(draft record 필드 + AnalysisJob 컬럼 직렬화 + Response 필드). rationale(rule-based 점수근거)에
LLM 검증 경고를 섞지 않아 관심사 분리. 스키마 필드 1개 추가(익숙한 패턴).

---

## E5. 검증 대상 필드

**선택: summary·recommendedFix·recommendedTests 3필드 전부** (합쳐서 한 번에 추출·대조). 셋 다 LLM 생성분.

---

## E6. 연동 지점

**선택: 별도 `ReportEvidenceVerifier` 컴포넌트 + `enrichWithLlmReport`에서 write 성공 후 호출.**
검증기는 순수 함수라 단위테스트 쉬움. 예외 안전(오류 시 빈 경고 반환해 분석 안 깨짐).

**연동 형태**: write 성공 → `withGeneratedReport`로 LLM 텍스트 적용 → verifier로 위반 목록 산출 →
`withEvidenceWarnings`로 draft에 부착(경고만, 텍스트는 유지). rule-based 폴백 경로엔 경고 없음(빈 리스트).

---

## E7. 테스트 전략

**선택**: 검증기 단위테스트 — (1) 근거 안 클래스/파일 언급→경고 없음, (2) 근거 밖 `OrderValidator`·`X.java`→
경고, (3) 단일 대문자 단어·일반 한글→오탐 없음(보수성), (4) 빈 근거·빈 텍스트 엣지. + 워커: 위반 시
evidenceWarnings 채워지고 LLM 텍스트는 유지. 외부 호출 없음.
